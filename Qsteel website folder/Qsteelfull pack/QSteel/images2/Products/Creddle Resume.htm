Known Issues

Exporting might fail for unknown reasons, but here are the known issues that we're working on currently:

  • Improper rendering for skills in multiple columns
  • No support two columns of employment information
  • Margins and spacing might not render exactly the same
  • Various issues with rendering tabs in Google Docs
  • Various issues with converting to the Google Docs Format

Web Font Embedding

Creddle will embed your chosen web fonts in the exported .docx file so that your fonts will display in Microsoft Word for Windows even if you or your recipient do not have those fonts installed. However, font embedding is not supported in Microsoft Word for Mac. Additionally, embedded fonts may not work properly in Microsoft Word Online or Google Docs. When this occurs, these applications use system default fonts instead.

If you need to target these other platforms with a stronger assurance that your resume will render with your chosen fonts, we have the following suggestions:

  1. Select a Standard Font rather than a Web Font
  2. Change the fonts in your .docx file using your word processing program
As we work to improve exporting to the .docx format, we will also be looking to add more options and make these workarounds as seamless as possible.

Note: Embedded fonts in .pdf format is supported on all platforms when using your browser to print your Creddle résumé as a .pdf file.

Resume Parsing

Your .docx file was exported with automated résumé parsing in mind. When submitting your Creddle résumé through a job applicant tracking system, we recommend using a .docx file rather than printing your résumé as a .pdf file for the best assurance that your résumé will be parsed properly.

DOCX vs. PDF

When choosing whether to submit your Creddle résumé as either a .docx file or a .pdf file, there can be a lot of opinions. We hope that these general guidelines help:

Exporting to .docx has a stronger assurance of being parsed properly by a machine. We recommend .docx files when any of the following is true:

  • When submitting through submission forms
  • When using standard fonts
  • When sure the recipient is using Microsoft Word for Windows to open .docx files

Exporting to .pdf has a stronger assurance of rendering Web fonts on all platforms. We recommend .pdf files when all of the following is true:

  • When sending to a person
  • When using Web fonts
  • When unsure if the recipient is using Microsoft Word for Windows to open .docx files

Help us improve!

Lastly, if your .docx file doesn't look right, help us improve our .docx exporting feature by sending us an email at contact@creddle.io - a link to your Creddle résumé and an attached .docx file will help us fix issues even faster.

We hope that even an early release our .docx exporting feature will help you get your Creddle résumé in the hands even more people than before (:
